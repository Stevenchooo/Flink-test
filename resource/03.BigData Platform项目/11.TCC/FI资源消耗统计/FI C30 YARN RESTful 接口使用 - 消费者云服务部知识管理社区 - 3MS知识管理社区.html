FI C30 YARN RESTful 接口使用
  • 编辑
    发表于：2016-04-06 15:51阅读：104评论：3

    要使用YARN的RESTful接口，需要先明确YARN RM的访问IP。

     

    在调用RESTful接口的设备上安装配套的FI客户端。

    并使用kinit进行kerberos认证。

    [code lang="bash"]
    kinit tcc
    [/code]

     然后使用curl工具调用RESTful接口：

    [code lang="bash"]
    curl -i --negotiate --insecure -u : https://10.51.30.17:26001/ws/v1/cluster/apps/application_1457586581950_6114
    [/code]

    可以得到如下结果：

     

    非常方便快速，比hdfs命令快了N倍。

    YARN的RESTful接口使用参考资料：

    http://hadoop.apache.org/docs/r2.4.1/hadoop-yarn/hadoop-yarn-site/WebServicesIntro.html

     

    注意：

    使用FI C30 YARN RESTful接口的时候，不能直接用admin用户，否则调用RESTful接口时会失败。

     

    使用RESTful接口前，需要新建一个用户，具有yarn生hdfs的相应权限。

    首先新建一个YARN的角色

    然后再建一个HDFS角色

    再新建一个用户，属于hadoop群组，并赋予刚刚新建的2种角色。

     

    此时，用新建的用户就能使用RESTful接口了。

    FI专家的解释是：admin用户是缺省的用户，但不是万能的用户。最好为每类业务都新建一个用户，这样业务功能才有保障。

     

    注意：新建yarn角色后，需要重启yarn服务。


    题外话：当新建完用户后，用admin也不报错了。


    附：YARN的WEB接口

     

     模块  Web Interface  说明  默认端口
     ResourceManager  https://rm_host:port/  主要包含了集群管理的Nodemanager的资源(cpu和内存)使用概况，在集群上运行的应用（提交/接收/运行/失败/终止等）概况，以及所有调度队列的使用概况。  26001
     NodeManager  https://nm_host:port/  主要包含了当前节点上的资源使用情况，当前节点上运行的应用以及相应的容器概况。同时还可以查看nodemanager的配置项，运行log等。 26010
     MapReduce JobHistoryServer  https://jhs_host:port/  主要包含了运行应用的历史信息，如应用的运行ID，时间，状态，日志等。  26014
